# 1974 프로젝트 — 문서 인덱스

Claude Code가 이 프로젝트에서 작업할 때 반드시 먼저 읽는 파일.
모든 상세 내용은 아래 **4개 문서로만** 관리한다. 작업 대상에 맞는 문서 하나만 열어서 본다.

| 문서 | 범위 |
|---|---|
| [docs/엔진.md](docs/엔진.md) | Sand Engine 전체 규칙 + 프로젝트 마스터/채널 규칙 + 폴더 구조 + 데이터 테이블 연동 + 성능 |
| [docs/골목길.md](docs/골목길.md) | 골목길(GOLMOK) 게임 — 캐릭터/배경/시그니처 FX |
| [docs/펌프.md](docs/펌프.md) | 펌프(PUMP) 게임 — 진화 시스템 |
| [docs/누리.md](docs/누리.md) | 누리(NURI) 게임 — 효과 목록 + 모듈 아키텍처 |

## 작업 원칙
- 새 게임/엔진 기능 작업 전 해당 문서를 먼저 읽는다.
- 작업이 끝나면 해당 문서의 "변경 기록" 섹션에 한 줄 추가한다. 새 파일을 만들지 않는다.
- 위 4개 외의 `.md` 문서를 새로 만들지 않는다. 옛 버전은 `backup/docs_legacy_20260620/`에 보관.
- `tang_sample/`은 별개의 참고용 샘플 프로젝트이며 이 문서 체계와 무관하다.

## 토큰 절약 원칙
- 큰 파일은 전체 Read 금지 — Grep으로 줄 번호 찾고 그 구간만 읽는다.
- 프리뷰 스크린샷이 1번 타임아웃되면 재시도하지 말고 `preview_eval`로 전환한다 (백그라운드 탭은 rAF가 멈춰 항상 실패함).
- 수정 하나당 핵심 동작 1~2개만 검증하고 끝낸다. 풀세트 재검증 안 함.
