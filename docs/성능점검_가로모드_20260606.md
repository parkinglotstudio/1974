# 가로(시네마) 성능 점검 — 엔진 리뷰

> 작성: Claude (Opus) · 2026-06-06
> 증상: 가로 시네마 해상도(1760×740)로 키우자 **프레임 급락**.
> 결론: 해상도 비례 비용 중 **CPU측(거대 캔버스 + readback 스톨)이 주범** → 엔진/게임에서 3가지 수정. GPU 패스는 2.6배라도 문제 아님.

---

## 찾은 원인 & 수정 (커밋됨)

### 1. ★ 레이어 캔버스 폭 과대 (가장 큼)
- `LayerSystem`이 캔버스 폭 = `viewWidth × mult`([4,8,1,8]). 가로 1760뷰 → **L1/L3 = 14080폭** (배경은 2433폭뿐인데).
- 매 프레임 clear/composite가 이 거대 캔버스를 처리 → 급락.
- **수정**: `LayerSystem.MAX_CW = 5120` 상한 캡. L1/L3 14080→5120 (**2.8배↓**). 세로엔 영향 거의 없음.
- ⚠️ 배경 월드폭이 5120 넘으면 `MAX_CW`도 올릴 것(안 그러면 스크롤 잘림).

### 2. ★ 근경림 readback 스톨 (_buildFxSrc)
- 근경 역광림 FX가 매 프레임 `getImageData`로 **풀폭 × 반해상 ≈ 1.14M** 픽셀을 GPU→CPU 읽음(스톨). 고해상도일수록 GPU 대기열이 길어 스톨이 더 비쌈.
- **수정**: 근경림 소스를 **반해상→쿼터해상**(`hw=W>>2, hh=H>>2`). readback **1.14M→0.41M (2.8배↓, 960×540보다도 작음)**, 림 JS 루프 4배↓.
- **캐릭터 영역 readback은 그대로 풀해상** 유지 → 캐릭터 화질 영향 없음. 근경 역광림만 살짝 더 부드러운 글로우(밤에만 나오는 서브틀 FX라 영향 미미).

### 3. 원경 크로스페이드가 L0를 비정적화
- 크로스페이드(_img 2장)가 L0를 `_static=false`로 만들어 **매 프레임 재래스터**.
- **수정**: `FAR_XFADE=false`(OFF) → 원경 정적 복귀. 켜려면 플래그만 ON(그땐 비용 감수).

## 남은 해상도 비례 비용 (현재 수용 가능)
- **L2(캐릭터) flush `putImageData`** 풀캔버스(1760×740≈5MB/프레임). 단순 대역폭(수백 MB/s) — GPU 대역폭 충분, 스톨 아님. 정 필요하면 dirty-rect로 캐릭터 bbox만 쓰게 최적화 가능(엔진 변경, 테스트 필요).
- **풀캔버스 GPU 패스**(lighting multiply·glow·fog·vignette·composite): 면적 2.6배지만 전부 GPU 합성 → 저렴.
- **전환(sandburst)**: `_tr` 중 풀캔버스 `getImageData`+픽셀루프+`putImageData`를 매 프레임(2초). **전환 순간만** 무거움. 거슬리면 G스텝 키우거나 `BG_TRANS_MS` 줄이기.

## 권장
1. 지금 **1760×740 그대로 테스트** — 위 3수정으로 평상시 비용이 960×540 수준 근처로 내려갔을 것.
2. 그래도 부족하면 순서대로: ① 해상도 한 단계↓(예 1536×648, 캐릭터 40%) ② 전환 비용 최적화 ③ L2 dirty-rect flush.
3. 근경림 쿼터해상이 너무 흐리면 `hw=W>>2`→`W>>1`(반해상)로 되돌리되 readback 비용 복귀.
