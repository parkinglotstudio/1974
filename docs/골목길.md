## 0. ⚠️ 절대 규칙 (다른 문서와 동일 — 가장 중요)

1. **커밋 안 한 작업은 사라질 수 있다.** tracked 파일(`GolmokGame.js` 등)을 `git checkout/restore/reset --hard`로 되돌리지 말 것. 의미 단위마다 **사용자에게 커밋 요청**.
2. **커밋·푸시·배포는 사용자가 "하라"고 할 때만.** 임의 금지.
3. **엔진 소스 비노출** — 배포는 `npm run build` → `dist/`(번들+압축, 소스맵 없음, `engine/`·`scripts/`·`samples/` 미포함). 자세한 건 §9.
4. **`dev_server.py` 손대지 말 것.**
5. **NO-TOUCH**: `.claude/`(워크트리), `.claire/`, `backup/`(읽기만), `node_modules/`.
6. **사용자가 직접 1차 테스트한다.** 코드/에셋 변경 후 `node --check`(JS) / `JSON.parse`(JSON) 구문검사만 하고 넘긴다. (단, 사용자가 스크린샷으로 버그를 보고하면 그때는 preview로 직접 확인하며 디버깅함 — 이번 세션에서도 그렇게 진행했다.)
7. **`games/golmok/samples/`(레퍼런스 GIF/원본, gitignore)는 로컬 전용** — 산출물(JSON)만 커밋.
8. **`docs/*.md`가 source of truth.** 작업 후 관련 문서 갱신.

## 3. 캐릭터 작업 — 무엇을 했나

### 3.1 변환 파이프라인 (확정, 변경 불필요)
- **`tools/build_female_char.py`** — 여성(1990) 캐릭터 빌드 스크립트. `python tools/build_female_char.py` 실행 시 `games/golmok/samples/*.gif`(로컬 전용)에서 `pixels/characters/ch01_gun.json`, `ch01_player.json` 재생성.
- 핵심 로직(`normalize_to_char_h`):
  - **스케일 기준 = "서있는 첫 프레임"의 머리~발 높이**(`CHAR_H=255`). union-bbox 높이로 잡으면 안 됨(과거 버그, §3.3 (A)).
  - `per_frame=True`(run/run_stop): 프레임별 자기 bbox로 크롭 후 스케일 — **폭이 캔버스(149px)를 넘어도 OK, 절대 폭 기준으로 추가 클램프하지 말 것** (했다가 키 절반 축소 회귀 2번 발생, §3.3 (F)).
  - `TORSO_ALIGN={'run','run_stop'}`: 달리기류는 **몸통 중심**(`_torso_cx`) 기준 정렬(팔다리만 클립, 몸통 안 튐).

### 3.2 이번 세션(2026-06-11)에서 새로 한 것
- **달리기 모션 추가**: `run`(12f, 20fps, loop) + `run_stop`(8f, 14fps, non-loop). 이동키 **2초 이상 홀드 → 달리기**(속도 ×1.8), 떼면 `run_stop` 재생 후 `idle`로 복귀.
  - 애니 플로우(`games/golmok/animator/ch01.anim.json`): `idle→walk→run→run_stop→idle` 전환 그래프.
  - `animations.csv`에 `run`/`run_stop` 행 추가(fps/loop override).
- **SandScroll 가장자리 띠 효과(`_renderSandScroll`/`_sandBand`) 완전 제거** — 코드/CSV/문서에서 모두 삭제. 좌우 FX는 `_renderEdgeFrame`(항상 켜진 검정 침식 프레임)만 남음.
- **화면 정지/재생 기능 추가** — `index.html`의 ⏸ 버튼 / **P 키**. `engine.stop()`/`start()` 토글. (디버깅·스크린샷용)

### 3.3 알려진 변환 함정 (요약 — 자세한 건 `안티그래비티_캐릭터애니_인수인계.md` §3)
- **(A)** 스케일은 "서있는 첫 프레임" 기준, union-bbox 기준 아님.
- **(D)** 플레이어 캔버스는 **149×259 고정**. 다운스트림(발 위치/카메라/groundY)이 이 크기를 전제 — 바꾸려면 `characters.csv`·`main_landscape/portrait.scene.json`·`GolmokGame.js` 전부 동기화 필요.
- **(E)** 엔진은 팔레트 idx10만 emissive로 봄 → `engine.glow.removeEmissiveIndex(10)` 호출 유지(밤 노란 halo 방지).
- **(F) [미해결] run/run_stop 뒷발 클리핑**: run 프레임 원본 폭이 최대 722px(스케일 후 ~284px)로 캔버스(149px) 초과 → run 12프레임 중 8개·run_stop 8프레임 중 다수가 좌/우 끝(`x=0` 또는 `x=148`)에서 발/팔이 잘림.
  - **시도한 해결**: 폭 기준 `sc` 추가 클램프 → **캐릭터 키가 절반(~120px)으로 축소되는 회귀** 발생(2026-06-11, 5f0045a에서 이미 한 번 발견된 문제와 동일). 즉시 원복.
  - **근본 해결**: `PLAYER_W`(149)를 ~200~290px로 넓혀야 함. 단 `pw/ph`가 `characters.csv`, `main_landscape.scene.json`, `main_portrait.scene.json`, `GolmokGame.js`(카메라 센터링·월드 바운드·groundY 계산, 총 4곳)에 참조되어 있어 동시 수정 필요.
  - **현재 상태**: 사용자가 "일단 보류, 다음 지시 대기"라고 해서 **미작업**. 키 정상 유지 우선, 발끝 약간 클립 허용.

## 4. 배경 작업 — 무엇을 했나 (2021 도시)

### 4.1 변환 파이프라인
- **`tools/split_bg_layers.py`** — 범용 근경/원경 분리 변환기 (메디안컷 적응형 팔레트). `{era}_1.png`(원경)→`{era}_far.json`, `{era}_1_1.png`(근경, 흰배경 실루엣)→`{era}_near.json`.
- **`tools/convert_2021_bg.py`** (2026-06-10) — 2021 맵 전용. 근경 1장(이미 알파있는 검정 실루엣) + 원경 2장(풀컬러, 원경2/원경7) → `2021_near.json`/`2021_far.json`/`2021_far7.json` (3748×960, 가로 씬 기준).
- **`tools/fix_far_alpha.py`** — `2021_far`/`2021_far7` 좌측 1~2px 투명 라인(원본 PNG 배경제거 흔적) 보정. 원경(L0)은 항상 화면을 덮어야 하므로 alpha를 강제 불투명으로 재양자화.
- **`tools/convert_near_new1.py`** (2026-06-11, 신규) — **사용자가 직접 제작한 색보존 근경 원본** `samples/2021_NEW_1.json`(frames/pixels 포맷, 4898×2010, 팔레트129, 57MB→43MB)을 읽어:
  1. sparse pixels → RGBA 이미지 복원
  2. 월드폭 3119×1280으로 리사이즈
  3. `fill_large_holes(min_area=1)`로 내부 투명 구멍 전부 메움(91,816px)
  4. `quantize_scanline(max_colors=160)`으로 scanline 재양자화 → `2021_near.json`
  - 결과: 불투명 36.8%, 평균RGB (8,10,12) — 어둡지만 창문 불빛 등 디테일 보존.
  - **이 변환의 입력 원본(`2021_NEW_1.json`, 43MB)은 `games/golmok/samples/`로 이동됨**(gitignore, 로컬 전용). `pixels/objects/`에는 절대 두지 말 것 — 배포 빌드에 그대로 포함되어 용량 폭증 원인이었음(§9).

### 4.2 원경 모래 전환(FAR_SWAP) ★시그니처
- **목적**: 월드 진행 중 **원경(하늘/스카이라인)만** 모래 효과로 `2021_far`→`2021_far7`로 교체. 근경/캐릭터는 불변.
- **트리거**: `cameraX >= (worldW - viewW) * 0.5`(월드 중간), 1회 래치, 인트로 끝난 뒤만, `onEnter`에서 래치 리셋(씬 재진입마다 재동작).
- **구현**: `GolmokGame.onRender(cameraX)`에서 composite 이전에 L0 캔버스에 직접 그림(정적 레이어라 entities.render가 덮어쓰지 않음). `_prepareFarSwap`이 `bg_far`의 asset이 정확히 `'2021_far'`일 때만 무장 → 다른 씬(2020 등)에 잘못 끼어들지 않음.
- 상수: `FAR_SWAP_ENABLED=true`, `FAR_SWAP_AT=0.5`, `FAR_SWAP_MS=1600`, `FAR_SWAP_OLD='2021_far'`, `FAR_SWAP_NEW='2021_far7'`.
- 자세한 블렌드 알고리즘은 `골목길_시그니처FX_수치.md` §6.5.

### 4.3 맵 등록/순환 (`maps.csv`)
- 현재 `cycle_order >= 0`(순환 대상)은 **`2021` 단 하나**. 나머지(`2020a/2020b/1960_1/2020_1/19870_2`)는 `cycle_order: -1`로 **순환 제외 + 코드/씬 어디서도 미사용** — `pixels/objects/`엔 파일이 남아있지만 배포 빌드에서는 자동 제외됨(§9).
- 새 맵을 순환에 추가하려면: ① `split_bg_layers.py`(또는 전용 변환기)로 `{id}_near`/`{id}_far` 생성 → `pixels/objects/` ② `maps.csv`에 행 추가(`cycle_order` 0 이상, `near_rim` 설정) ③ 씬 JSON에 엔티티 배치. **`build.mjs`는 따로 손댈 필요 없음**(자동 필터링).

## 2. 애니메이션 시스템 아키텍처 (유니티 Animator 사상)

3계층으로 분리돼 있다. **이 구조를 지킬 것.**

### (a) `engine/entity/AnimationStateMachine.js` — 클립 재생기 (asm)
- `states = { name: { frames:[절대인덱스...], loop, fps } }`
- `setState(name)`(같은 상태면 무시), `tick(now)`(프레임 진행), `getCurrentFrame()`, `isDone()`(non-loop 완료), `restart()`(0프레임부터).
- 내부: `_stateFrameIdx`, `_lastTick`, `_done`.
- 엔티티가 매 프레임 `asm.tick()` 호출 → 현재 프레임 인덱스 결정.

### (b) `engine/anim/AnimatorController.js` — 플로우(상태 그래프)
- `load(def)` / `setParam` / `setBool` / `trigger` / `update()`.
- `def = { default, parameters:[{name,type:'float'|'bool'|'trigger',default}], transitions:[ {from,to,conditions:[{p,op,v}],hasExitTime} ] }`
- `op`: `> >= < <= == != trigger`. `from:'*'` = Any State. `hasExitTime:true` = 클립 끝나야 전환.
- 게임 로직은 **파라미터만** 세팅, 어떤 클립이 재생될지는 컨트롤러가 결정.

### (c) 스프라이트 `stateDef` (JSON 안) — 상태 정의 데이터
- `{ "sprite":"...", "states": { "idle": {"frames":[...],"loop":true,"fps":5}, ... } }`
- 엔티티 생성 시 `stateDef` 넘기면 `entity.asm.load(stateDef)`.

### (d) `animations.csv` — fps/loop **override**
- `char_id,state,frames,fps,loop,notes`
- 로드 시 `this._player.asm.states[state].fps/loop`를 CSV값으로 **덮어씀**(존재하는 상태만).
- ⚠️ **프레임 데이터는 스프라이트 stateDef가 진실.** CSV의 `frames` 칸은 참고용(개수 메모)일 뿐. fps/loop만 실제 반영.

### 현재 캐릭터 구성
- **플레이어**(`ch01_player`, 149×259): `idle`(루프) + `walk`(루프).
- **총**(`ch01_gun`, 동적 크기): `atk_start`(준비, non-loop) → `atk_active`(발사, 루프) → `atk_recover`(회수/내리기, non-loop).
- 총 플로우(`GolmokGame._loadGun`): `atk_start→atk_active`(hasExitTime), `atk_active→atk_recover`(firing==false). **공격대기(standby)는 제거됨** — 회수 끝나면 게임 코드가 바로 idle로 전환(아래 §6).

## 3. ★ GIF → 스프라이트 변환 파이프라인 (핵심 노하우)

캐릭터 모션은 **그린스크린 GIF 여러 개 → 단일 다상태 스프라이트 JSON**으로 만든다.

### 두 변환기
- **`tools/gif_to_sprite.py`** — 범용. **동일 해상도** 클립들을 union-bbox로 정합. (남자 캐릭터가 이걸로 만들어짐.)
- **`tools/build_female_char.py`** — 여성 캐릭터용. **해상도가 다른 클립 혼합**(여성 720p + 남자 walk 270p)에 대응. 총은 union-bbox, 플레이어는 클립별 정규화+발밑 정렬.

### 파이프라인 단계 (gif_to_sprite.py 기준)
1. 각 GIF 프레임 RGBA 로드.
2. **그린스크린 제거(dekey)** + 디스필: `g - max(r,b) ≥ GREEN_DOM(38)` & `g ≥ GREEN_MIN(70)` → 투명. 가장자리 초록끼는 despill.
3. **마커 앵커 추출**(`extract_anchors`): 소스에 찍힌 고유색 점을 부착점으로 기록 후 제거.
   - 마젠타(255,0,255)=**muzzle(총구)** · 시안(0,255,255)=**hand** · 노랑(255,255,0)=**eject(탄피구)**. (2~3px 점, 다운스케일 생존)
   - 프레임별 `frame.anchors[name]=[x,y]`로 저장 → 렌더에서 정밀 위치(`GolmokGame._frameAnchor`)로 사용.
4. **공통 bbox 크롭** → 상태 간 캐릭터 위치 정합(안 튐). *동일 해상도 전제.*
5. **균일 스케일** — ⚠️ **함정 주의**(아래).
6. 클립별 후처리(`fx`): `'clean'`=작은 분리 blob(탄피) 제거 · `'white2black'`=흰색→검정.
7. **통합 메디안컷 양자화**(`quantize_all`) — 전 프레임 1팔레트(프레임마다 색 안 흔들림). `palette[0]='transparent'`.
8. `stateDef`(상태=프레임범위/fps/loop) 구성 → JSON 출력.

### ⚠️ 변환 함정 (과거에 실제로 터진 것들 — 반드시 숙지)
- **(A) 스케일 기준 = “서있는 캐릭터(첫 프레임) 머리~발 높이”로 잡을 것. union-bbox 높이로 잡지 말 것.**
  - 과거 버그: union-bbox에는 머즐플래시·탄피 등 캐릭터보다 위로 튄 효과가 섞여 bbox가 커짐 → 그 기준으로 스케일하면 **캐릭터가 작아져 플레이어와 키가 안 맞음**(236 vs 259 사건).
  - 해결: `ref = clips[0][0].getbbox()`(준비/idle 첫 프레임, 서있는 키) → `scale = TARGET_H / ref_h`. (`gif_to_sprite.py`/`build_female_char.py` 둘 다 이렇게 돼 있음. 유지할 것.)
- **(B) 발사/회수 포즈가 짧은 건 “숙인 포즈”라 정상.** bbox 높이가 작아도 스케일은 동일(같은 ref). 작아 보인다고 다시 스케일하면 오히려 커진다.
- **(C) 짧은 발사 루프(예: 3프레임) 머즐 검출.** `_loadGun`이 `atk_active.frames[3]`(4번째 프레임)을 머즐플래시 샘플에 쓰는데, 프레임이 3개뿐이면 `[3]`이 undefined. → **`frames[Math.min(3, len-1)]`로 클램프**돼 있고 `_muzzleLocal` 기본값도 둠. 새 발사 클립이 짧으면 이 처리에 의존.
- **(D) 스프라이트 크기 = 엔티티 크기.** 총은 `pw:g.width, ph:g.height`(동적, 발 앵커로 정합)라 자유. **플레이어는 149×259 고정 권장** — 다운스트림(발 위치/바운드/groundY)이 이 크기를 전제. 다른 크기로 만들면 정합 깨짐.
- **(E) 발광 인덱스.** 엔진은 팔레트 **idx10**만 emissive로 본다. 캐릭터엔 발광이 없으므로 게임이 `engine.glow.removeEmissiveIndex(10)` 호출(밤에 노란 halo 방지). 새 팔레트도 이 호출로 보호됨 — 손대지 말 것.

## 5. ★ 시그니처 = 엔진 정체성 (가장 중요 — 강조)

> **이 게임의 정체성은 “픽셀이 모래로 부서졌다 다시 형성되는” 연출.** 단순 크로스페이드가 아니라 **모래 블렌드/코팅**이 우리만의 시그니처다. 캐릭터 애니의 전환·등장·공격 연출은 전부 이 결을 따라야 한다.
>
> 대원칙(기존 `GOLMOK_SIGNATURE_FX.md`): **“시그니처 = 엔진 기능. 로직은 `engine/`, 게임은 수치/색만.”** 단, 캐릭터 모래 코팅은 현재 `GolmokGame.js`에 게임측으로 들어가 있음(추후 엔진화 여지).

### 핵심 시그니처 연출 (구현 위치 = `GolmokGame.js`)
- **`_renderSandCoat` (모래 코팅 전환) ★캐릭터 애니 시그니처**
  - 동작 전환 시 “모래 띠가 몸을 한 방향으로 쓸고 지나가며” 모래가 생성→안착→사라짐.
  - 방향 통일: **라이즈(공격 시작)=뒤→앞**, **회수/복귀(→idle)=앞→뒤**.
  - 모래는 **현재 동작 실루엣 위에** 얹힌다(엔티티는 솔리드 100% 렌더 → 깜박임 방지, 모래는 오버레이). `src`=출발 포즈 픽셀에서 날아와 안착(`fly`), 띠 양끝 `sin` 페이드, `surge`로 진행방향 출렁.
  - `_startCoat(mode,bodyEntity,src,ms,onDone)`로 시작. `mode='raise'|'lower'`.
  - 공격 흐름: `_startAttack`(준비 중간프레임부터 + raise 코팅) → 발사(`_spawnShot` 머즐버스트/트레이서/반동/글로우) → 회수 끝 → **idle로 전환 + idle 위 lower 코팅**.
- **`_renderCharReveal`** — 낮↔밤 캐릭터 색 reveal(밤엔 배경톤으로 부드럽게 어두워짐). 수치는 `GOLMOK_SIGNATURE_FX.md`.
- **`_renderGunLight`** — 공격 중 총에 **플레이어 reveal과 동일 공식**으로 낮밤 톤 적용(검정 아님, 배경톤). `_lastAvgLum` 재사용.
- **`_renderCharRim` / `_renderNearRim`** — 캐릭터/근경 외곽 역광 림(픽셀별 배경색 env).
- **`_renderIntro`** — 진입 시 모래가 사람 실루엣을 형성 후 실제 캐릭터로 드러남.
- 낮↔밤 사이클 `k = 0.5 - 0.5*cos(dayT·2π/60)`(60초). 배경 4장 순환 + `_bgTone` 자동 매칭.

### 연출 작업 시 지킬 것
- 새 전환/등장/특수 모션은 **크로스페이드 금지 → 모래 코팅/디졸브 결**로.
- 깜박임 방지: 캐릭터 본체는 엔티티가 솔리드로 그리고, 효과는 그 **위 오버레이**로.
- 방향성 통일(생성·소멸 방향) 유지.
- 값 바꾸면 `docs/GOLMOK_SIGNATURE_FX.md` 갱신.

## 6. 이동/공격 상태 & 입력 (GolmokGame.js)

- **이동 잠금:** 공격/블렌드(비이동 모션) 중엔 이동키 무시 — `if (this._attacking || this._blend) { dx=0; }`. 이동은 idle/walk에서만. (특수 모션이 이동을 다시 허용하려면 그때 분기.)
- **공격:** `F`키 → `_startAttack`. 흐름 = 준비→발사(누르는 동안 루프)→회수→**바로 idle**(공격대기 없음). 회수 끝 처리: `asm.current==='atk_recover' && asm.isDone()` → 총 숨김/플레이어 보임/`idle` 0프레임 restart + lower 코팅.
- **총 정합:** `_positionGun`(발 앵커 `_gunAnchor`로 플레이어 발치에 맞춤, flipX 미러). 머즐 위치 `_muzzlePos`(있으면 `frame.anchors.muzzle`, 없으면 `_muzzleLocal` 휴리스틱).

## 현재 상태 한눈에 (2026-06-03 복구 완료)
- **낮↔밤 사이클: ON** — `k = 0.5 - 0.5*cos(dayT * 2π/60)`, 60초 주기.

  - 낮(k=0) → **100%** 원본색 / 밤(k=1) → **20%(어두운 배경)~35%(밝은 배경 앞)**.
  - `reveal = 1 - (1-nightReveal)*k`, `nightReveal = 0.20 + 0.15*min(1, avgLum*1.6)`.
- 캐릭터 rim = **게임측 `_renderCharRim` 단일** (엔진 rim은 OFF — lightDir 반대측 엣지에도 그려 방향이 뭉개져서).

    - (이전 boost 2.0~2.3은 밝은 배경 앞에서 형광 노란 줄무늬가 됨 → 낮춤)
  - 결과: 낮·황혼(밝음)엔 림 없음 → 깊은 밤(밝기 20%)에서만 은은한 역광. (캐릭터가 밝으면 림 없이도 보이므로)

# golmok 게임 데이터 테이블 (CSV)

**원칙: 엔진 = 로직(코드) / 게임 = 데이터(CSV).**
코드에 숫자를 박지 않고 이 표에서 읽는다. 나중에 에디터 툴이 이 CSV만 편집하면 게임이 바뀐다.

## 로딩 흐름
```
games/golmok/data/*.csv
   ↓ main.js init()  (engine/data/Csv.js 로더)
engine.data = { maps, characters, animations, fx, targets, projectiles }
   ↓ GolmokGame._loadConfig(engine)   ← onEnter에서 1회 (targets/projectiles는 _loadTargets/_spawnMissile에서 직접 참조)
모듈 상수(let) 덮어쓰기 + this._p(인라인 수치) 채움 + 애니 fps override
```
CSV 로드 실패 시 코드 내 fallback 기본값으로 동작(게임은 안 죽음).

## 파일

| 파일 | 행 = | 핵심 열 |
|---|---|---|
| `maps.csv` | 배경 1장 | id, asset, world_w/h, **cycle_order**(순환 순서, -1=제외), **near_rim**(맵별 근경 역광 배율, 밝은 야경=낮게) |
| `characters.csv` | 캐릭터 1명 | id, asset, pw/ph, spawn_x/y, **move_speed**, layer |
| `animations.csv` | 캐릭터 상태 | char_id, state, frames, **fps**, loop |
| `fx_params.csv` | 파라미터 1개 | key, **value**, min, max(슬라이더용), desc |
| `targets.csv` | 피격 타겟 1개 | id, asset, assetCategory, pw/ph, spawn_x/y, **hp**(0=무한), **hit_ms**(피격 시 흩어짐↔재조립 시간) |
| `projectiles.csv` | 발사체 타입 1개 | id, **speed**(px/s), **arc_height**(포물선 정점), size, color, life_ms |

## RPG 기본 시퀀스 (공격 ↔ 피격)
- `_loadTargets()` (onEnter) — `targets.csv` 각 행을 `pixels/<assetCategory>/<asset>.json`에서 로드해 고정 엔티티로 배치 (`this._targets`).
- `_spawnShot()` → `_findTargetAhead(dir)`로 사격 방향의 사거리(500px) 내 타겟 탐지 → `_spawnMissile()`이 `projectiles.csv`(`missile01`) 파라미터로 포물선 발사체 생성.
- 발사체가 타겟에 도달하면 `_hitTarget()` — 플레이어 idle↔사격 전환과 동일한 픽셀무브 톤(`_renderPixelMove`)으로 흩어짐→재조립(`hit_ms`).
- 새 타겟/발사체 타입 추가 시 코드 수정 없이 `targets.csv`/`projectiles.csv`에 행만 추가.

## 역할 분담 (씬 vs 테이블)
- **테이블(CSV)** = 타입·스탯·튜닝 (이동속도, 애니 fps, FX 수치, 맵 순환)
- **씬(scene.json)** = 배치 (어떤 맵·캐릭터를 어디 좌표에 놓는가). 좌표는 씬이 우선.
- **픽셀 JSON(pixels/)** = 실제 그림 자산 (CSV 아님)

## fx_params.csv 그룹
- `daynight.*` 낮↔밤 사이클(주기, 최저 밝기)
- `rim.*` 캐릭터 림 라이트(페이드 임계, 세기, floor, 두께, 방향가중)
- `nearrim.*` **근경 역광 림**(낮/밤 세기) — 맵별 배율은 `maps.csv near_rim`
- `reveal.*` 캐릭터 색 리빌 곡선
- `light.*` 전역 조명: ambient/vignette/fog, **glow_max(0.5)**, **bloom_threshold(205, 밝은 불빛만 번짐)**
- `bg.*` 배경 전환/모래알
- `intro.*` 진입 인트로
- `sand.*` 좌우 이동 모래 스크롤

## 밤 = 콘트라스트 (핵심 룰)
밤은 균일하게 어둡게가 아니라 **어두운 데 더 어둡게(ambient×k) + 밝은 데 더 밝게(glow×k)**.
밝은 야경 맵(2020)은 `bloom_threshold`↑·`near_rim`↓로 과블룸·네온윤곽을 막는다. 자세한 룰은 `docs/ENGINE_CAPABILITIES.md`.

## 값 바꾸는 법
CSV의 `value` 열만 수정 → 새로고침. 코드 수정 불필요.
(min/max는 현재 코드가 강제하진 않음 — 향후 에디터 슬라이더 범위용 메타.)
