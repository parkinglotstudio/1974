# 네모세모동그라미 (Runner) — 비주얼 브리프 (VISUAL BRIEF)
> **대상:** 단이(안티그래비티) 비주얼팀 (Dani(Antigravity) Art & Design)
> **담당:** 단이 (비주얼 디렉팅 & AI 비서)
> **최종 갱신:** 2026-05-24

---

## 1. 비주얼 제약 조건 (Hard Constraints)

우리 프로젝트인 **모래엔진(Sand Engine)**과의 완벽한 호환 및 프리미엄 레트로 미학을 달성하기 위해 아래 세 가지 기술 제약을 엄격히 준수해야 합니다.

1.  **해상도:** `480 × 270` (16:9 픽셀 와이드 스크린)
2.  **화면 스케일:** 1px 단위가 뭉개지지 않고 정확히 정수배(integer scale)로 구현되는 **픽셀 퍼펙트 (Pixel-perfect)** 렌더링.
3.  **팔레트 제한:** **최대 12색** 단일 팔레트 구성 (`/assets/palettes/palette_runner.json`). 색상 수의 제한은 레트로 감성과 디더링 기법의 미학을 극대화합니다.

---

## 2. 무드보드 및 비주얼 테마 제안 (3가지 시안)

오빠(기획자)의 "1974 범띠 — 주판에서 AI까지" 프로젝트 방향성에 맞추어, 80년대 말~90년대 초의 디지털 여정을 장식할 3가지 비주얼 컨셉을 제안합니다.

````carousel
### [시안 A] 아날로그 브라운 CRT (따뜻한 레트로 종이/목재 톤) ★ 단이 추천!
- **설명:** 주판(Ch.2)의 동양적이고 나무적인 느낌과 80년대 후반의 따뜻한 CRT 모니터 감성을 연결하는 테마. 부드럽고 친근한 레트로 스타일.
- **주요 기법:** 은은한 도트 디더링(Dithering)을 이용한 명암 조절, 클래식한 픽셀 마감.
- **색상 구성:**
  - *배경 & 지형:* 따뜻한 모래 베이지, 카키, 에스프레소 브라운, 황토색
  - *동그라미 (부스트):* 일렉트릭 터콰이즈 블루 (Smear 이펙트에 환상적!)
  - *세모 (슬로우):* 러스트 오렌지 레드
- **비주얼 예시:**
  ```
  [ L0 원경: 부드럽게 디더링된 저녁 노을과 지평선 ]
  [ L1 근경: 실루엣 처리된 옛날 전신주와 들판 ]
  [ L2 지면: 황토색과 브라운이 격자로 교차하는 징검다리 형태 ]
  ```

<!-- slide -->

### [시안 B] DOS 앰버 모니터 (허큘리스 모노크롬 감성)
- **설명:** 90년대 초 흑백 및 황동색 단색 컴퓨터 화면(Hercules Graphics)을 모티브로 한 컨셉. 아주 독특하고 하이엔드한 픽셀 감성을 제공.
- **주요 기법:** 단색의 명도 단계만을 조절하는 샌드 이펙트(Sand Effect)와 하프톤 디더링 극대화.
- **색상 구성:**
  - *메인 톤:* 다크 세피아, 앰버(황동색), 브라이트 골드 옐로우
  - *포인트:* 오렌지-옐로우 팔레트 안에서 3~4가지 톤 조절
- **비주얼 예시:**
  ```
  [ L0 원경: DOS 부팅 화면을 오마주한 메트릭스 격자선 ]
  [ L1 근경: 도트로 촘촘히 묘사된 8비트 구름 ]
  [ L2 지면: 앰버 모니터의 스캔라인이 흘러가는 단단한 바위 지형 ]
  ```

<!-- slide -->

### [시안 C] 네온 텍스처 그리드 (DOS C:\> 사이버 공간)
- **설명:** DOS 컴퓨터 세계관 속 "가상 현실 서바이벌"을 현대적인 디자이너 아케이드로 재해석한 컨셉.
- **주요 기법:** 네온 컬러의 와이어프레임 라인, 선명한 하이라이트 글로우 도트.
- **색상 구성:**
  - *배경:* 매트 블랙, 딥 네이비
  - *캐릭터 & 아이템:* 형광 그린, 네온 시안, 일렉트릭 마젠타
- **비주얼 예시:**
  ```
  [ L0 원경: 멀리 보이는 메가 시티의 기하학 와이어프레임 빌딩 ]
  [ L1 근경: 지나가는 데이터 케이블선과 신호 파티클 ]
  [ L2 지면: 빛나는 네온 그리드가 깔린 미끄러운 바닥 ]
  ```
````

## 3. 에셋 드로잉 명세 (Sprite Spec)

코드로 그리던 프로토타입 도형들을 대체할 픽셀 스프라이트 제작 명세입니다.

### 1) 플레이어 캐릭터 (Shape Character)
캐릭터는 기본 크기 **24 × 24 px** 박스 내부에서 디자인하되, 각 형태별로 직관적인 모션을 부여합니다.

*   **Square (네모):** 단단하고 클래식한 8비트 로봇 혹은 칩 스타일.
    *   *애니메이션:* Idle (1프레임), Run (4프레임 - 다리가 데굴데굴 구르는 느낌), Jump/Fall (각 1프레임)
*   **Circle (동그라미 - 부스트):** 에어로다이내믹한 구체 형태.
    *   *애니메이션:* Spin-Roll (4프레임 회전), Boost Trail (잔상 픽셀 세트)
*   **Triangle (세모 - 슬로우):** 아래가 무거운 피라미드 형태 혹은 경고판 스타일.
    *   *애니메이션:* Heavy Slide (2프레임 - 땅에 살짝 닿아 미끄러지며 먼지를 뿜는 연출)

### 2) 아이템 및 장애물 에셋
*   **코인 (Time Add):** 회전하는 8 × 8 px 픽셀 주판알 또는 금화 (4프레임 루프).
*   **파란 동그라미 (Boost):** 반짝이는 일렉트릭 블루 구체 (12×12 px).
*   **빨간 세모 (Slow):** 위험 경고 표시를 담은 붉은 세모 크리스탈 (12×12 px).

### 3) 지형 타일 및 배경 레이어
*   **Layer 2 (지면):** 캐릭터가 디딜 윗면은 경계가 확실하고(Solid), 내부 타일은 자연스럽게 반복(Tiling)되는 흙/돌/모래 텍스처 패턴.
*   **Layer 1 (근경):** 지면 바로 뒤에서 60% 스크롤 속도로 흘러가는 레트로한 풀숲 또는 컴퓨터 부품 잔해 실루엣.
*   **Layer 0 (원경):** 가장 느린 속도로 흘러가며 게임의 무드를 지배하는 하늘, 구름, 혹은 거대한 픽셀 노을.

## 4. 납품 및 워크플로우 (Deliverable Workflow)

1.  **비주얼 컨셉 확정:** 오빠(기획자)와 논의하여 위 3가지 시안 중 하나를 확정합니다.
2.  **픽셀 제작:** 브라우저에서 `http://localhost:9191`로 접속하여 모래엔진 비주얼 에디터(`sand_engine.html`)를 실행하고, 내부의 픽셀 아트 툴을 켭니다.
3.  **자동 저장:** 에디터로 작업물을 그리고 저장하면 자동으로 `PixelJSON` 포맷으로 변환되어 `/assets/pixels/{asset-name}.json`으로 보관됩니다.
4.  **엔진 연동:** 납품된 JSON은 단이(안티그래비티)와 클로드가 코드에 불러와 실시간 렌더링되도록 결합합니다. (비주얼팀은 드로잉에만 집중할 수 있게 지원합니다!)
