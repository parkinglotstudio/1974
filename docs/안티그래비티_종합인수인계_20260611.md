## 0. ⚠️ 절대 규칙 (다른 문서와 동일 — 가장 중요)

1. **커밋 안 한 작업은 사라질 수 있다.** tracked 파일(`GolmokGame.js` 등)을 `git checkout/restore/reset --hard`로 되돌리지 말 것. 의미 단위마다 **사용자에게 커밋 요청**.
2. **커밋·푸시·배포는 사용자가 "하라"고 할 때만.** 임의 금지.
3. **엔진 소스 비노출** — 배포는 `npm run build` → `dist/`(번들+압축, 소스맵 없음, `engine/`·`scripts/`·`samples/` 미포함). 자세한 건 §9.
4. **`dev_server.py` 손대지 말 것.**
5. **NO-TOUCH**: `.claude/`(워크트리), `.claire/`, `backup/`(읽기만), `node_modules/`.
6. **사용자가 직접 1차 테스트한다.** 코드/에셋 변경 후 `node --check`(JS) / `JSON.parse`(JSON) 구문검사만 하고 넘긴다. (단, 사용자가 스크린샷으로 버그를 보고하면 그때는 preview로 직접 확인하며 디버깅함 — 이번 세션에서도 그렇게 진행했다.)
7. **`games/golmok/samples/`(레퍼런스 GIF/원본, gitignore)는 로컬 전용** — 산출물(JSON)만 커밋.
8. **`docs/*.md`가 source of truth.** 작업 후 관련 문서 갱신.

## 3. 캐릭터 작업 — 무엇을 했나

### 3.1 변환 파이프라인 (확정, 변경 불필요)
- **`tools/build_female_char.py`** — 여성(1990) 캐릭터 빌드 스크립트. `python tools/build_female_char.py` 실행 시 `games/golmok/samples/*.gif`(로컬 전용)에서 `pixels/characters/ch01_gun.json`, `ch01_player.json` 재생성.
- 핵심 로직(`normalize_to_char_h`):
  - **스케일 기준 = "서있는 첫 프레임"의 머리~발 높이**(`CHAR_H=255`). union-bbox 높이로 잡으면 안 됨(과거 버그, §3.3 (A)).
  - `per_frame=True`(run/run_stop): 프레임별 자기 bbox로 크롭 후 스케일 — **폭이 캔버스(149px)를 넘어도 OK, 절대 폭 기준으로 추가 클램프하지 말 것** (했다가 키 절반 축소 회귀 2번 발생, §3.3 (F)).
  - `TORSO_ALIGN={'run','run_stop'}`: 달리기류는 **몸통 중심**(`_torso_cx`) 기준 정렬(팔다리만 클립, 몸통 안 튐).

### 3.2 이번 세션(2026-06-11)에서 새로 한 것
- **달리기 모션 추가**: `run`(12f, 20fps, loop) + `run_stop`(8f, 14fps, non-loop). 이동키 **2초 이상 홀드 → 달리기**(속도 ×1.8), 떼면 `run_stop` 재생 후 `idle`로 복귀.
  - 애니 플로우(`games/golmok/animator/ch01.anim.json`): `idle→walk→run→run_stop→idle` 전환 그래프.
  - `animations.csv`에 `run`/`run_stop` 행 추가(fps/loop override).
- **SandScroll 가장자리 띠 효과(`_renderSandScroll`/`_sandBand`) 완전 제거** — 코드/CSV/문서에서 모두 삭제. 좌우 FX는 `_renderEdgeFrame`(항상 켜진 검정 침식 프레임)만 남음.
- **화면 정지/재생 기능 추가** — `index.html`의 ⏸ 버튼 / **P 키**. `engine.stop()`/`start()` 토글. (디버깅·스크린샷용)

### 3.3 알려진 변환 함정 (요약 — 자세한 건 `안티그래비티_캐릭터애니_인수인계.md` §3)
- **(A)** 스케일은 "서있는 첫 프레임" 기준, union-bbox 기준 아님.
- **(D)** 플레이어 캔버스는 **149×259 고정**. 다운스트림(발 위치/카메라/groundY)이 이 크기를 전제 — 바꾸려면 `characters.csv`·`main_landscape/portrait.scene.json`·`GolmokGame.js` 전부 동기화 필요.
- **(E)** 엔진은 팔레트 idx10만 emissive로 봄 → `engine.glow.removeEmissiveIndex(10)` 호출 유지(밤 노란 halo 방지).
- **(F) [미해결] run/run_stop 뒷발 클리핑**: run 프레임 원본 폭이 최대 722px(스케일 후 ~284px)로 캔버스(149px) 초과 → run 12프레임 중 8개·run_stop 8프레임 중 다수가 좌/우 끝(`x=0` 또는 `x=148`)에서 발/팔이 잘림.
  - **시도한 해결**: 폭 기준 `sc` 추가 클램프 → **캐릭터 키가 절반(~120px)으로 축소되는 회귀** 발생(2026-06-11, 5f0045a에서 이미 한 번 발견된 문제와 동일). 즉시 원복.
  - **근본 해결**: `PLAYER_W`(149)를 ~200~290px로 넓혀야 함. 단 `pw/ph`가 `characters.csv`, `main_landscape.scene.json`, `main_portrait.scene.json`, `GolmokGame.js`(카메라 센터링·월드 바운드·groundY 계산, 총 4곳)에 참조되어 있어 동시 수정 필요.
  - **현재 상태**: 사용자가 "일단 보류, 다음 지시 대기"라고 해서 **미작업**. 키 정상 유지 우선, 발끝 약간 클립 허용.

## 4. 배경 작업 — 무엇을 했나 (2021 도시)

### 4.1 변환 파이프라인
- **`tools/split_bg_layers.py`** — 범용 근경/원경 분리 변환기 (메디안컷 적응형 팔레트). `{era}_1.png`(원경)→`{era}_far.json`, `{era}_1_1.png`(근경, 흰배경 실루엣)→`{era}_near.json`.
- **`tools/convert_2021_bg.py`** (2026-06-10) — 2021 맵 전용. 근경 1장(이미 알파있는 검정 실루엣) + 원경 2장(풀컬러, 원경2/원경7) → `2021_near.json`/`2021_far.json`/`2021_far7.json` (3748×960, 가로 씬 기준).
- **`tools/fix_far_alpha.py`** — `2021_far`/`2021_far7` 좌측 1~2px 투명 라인(원본 PNG 배경제거 흔적) 보정. 원경(L0)은 항상 화면을 덮어야 하므로 alpha를 강제 불투명으로 재양자화.
- **`tools/convert_near_new1.py`** (2026-06-11, 신규) — **사용자가 직접 제작한 색보존 근경 원본** `samples/2021_NEW_1.json`(frames/pixels 포맷, 4898×2010, 팔레트129, 57MB→43MB)을 읽어:
  1. sparse pixels → RGBA 이미지 복원
  2. 월드폭 3119×1280으로 리사이즈
  3. `fill_large_holes(min_area=1)`로 내부 투명 구멍 전부 메움(91,816px)
  4. `quantize_scanline(max_colors=160)`으로 scanline 재양자화 → `2021_near.json`
  - 결과: 불투명 36.8%, 평균RGB (8,10,12) — 어둡지만 창문 불빛 등 디테일 보존.
  - **이 변환의 입력 원본(`2021_NEW_1.json`, 43MB)은 `games/golmok/samples/`로 이동됨**(gitignore, 로컬 전용). `pixels/objects/`에는 절대 두지 말 것 — 배포 빌드에 그대로 포함되어 용량 폭증 원인이었음(§9).

### 4.2 원경 모래 전환(FAR_SWAP) ★시그니처
- **목적**: 월드 진행 중 **원경(하늘/스카이라인)만** 모래 효과로 `2021_far`→`2021_far7`로 교체. 근경/캐릭터는 불변.
- **트리거**: `cameraX >= (worldW - viewW) * 0.5`(월드 중간), 1회 래치, 인트로 끝난 뒤만, `onEnter`에서 래치 리셋(씬 재진입마다 재동작).
- **구현**: `GolmokGame.onRender(cameraX)`에서 composite 이전에 L0 캔버스에 직접 그림(정적 레이어라 entities.render가 덮어쓰지 않음). `_prepareFarSwap`이 `bg_far`의 asset이 정확히 `'2021_far'`일 때만 무장 → 다른 씬(2020 등)에 잘못 끼어들지 않음.
- 상수: `FAR_SWAP_ENABLED=true`, `FAR_SWAP_AT=0.5`, `FAR_SWAP_MS=1600`, `FAR_SWAP_OLD='2021_far'`, `FAR_SWAP_NEW='2021_far7'`.
- 자세한 블렌드 알고리즘은 `골목길_시그니처FX_수치.md` §6.5.

### 4.3 맵 등록/순환 (`maps.csv`)
- 현재 `cycle_order >= 0`(순환 대상)은 **`2021` 단 하나**. 나머지(`2020a/2020b/1960_1/2020_1/19870_2`)는 `cycle_order: -1`로 **순환 제외 + 코드/씬 어디서도 미사용** — `pixels/objects/`엔 파일이 남아있지만 배포 빌드에서는 자동 제외됨(§9).
- 새 맵을 순환에 추가하려면: ① `split_bg_layers.py`(또는 전용 변환기)로 `{id}_near`/`{id}_far` 생성 → `pixels/objects/` ② `maps.csv`에 행 추가(`cycle_order` 0 이상, `near_rim` 설정) ③ 씬 JSON에 엔티티 배치. **`build.mjs`는 따로 손댈 필요 없음**(자동 필터링).

  - `cycle_order: -1`(미사용) 맵 5쌍 + 완전 미참조 `2020long_near.json` = 약 67.5MB가 빌드에 그대로 포함.
- **조치**:
  1. `2021_NEW_1.json` → `samples/`로 이동(§4.1), `convert_near_new1.py` 경로 수정.
  2. `2020long_near.json`(14MB, 완전 미참조) 삭제.
  3. `build.mjs`를 수정해 `pixels/objects/`는 **`maps.csv`에서 `cycle_order >= 0`인 맵 접두어 파일만** 포함하도록 자동 필터링 (`pixels/characters/`는 전부 포함).
  4. 결과: `dist` **150MB → 84MB**. 로컬 프리뷰로 정상 동작 확인(2021 맵+캐릭터 렌더 OK, 콘솔 에러 없음).
  5. `app.*.min.js`에 `engine/` 경로·`.map` 파일 없음 재확인(엔진 비노출 유지).
  6. `main`에 커밋·푸시 완료(`195ddee`) → Vercel 자동 재배포 트리거.
- **앞으로 새 맵 추가 시**: `maps.csv`의 `cycle_order`만 올바르게 설정하면 `build.mjs`가 자동으로 처리. 단, **변환 원본/중간 파일은 절대 `pixels/objects/`에 두지 말 것** — `samples/`(gitignore)에.
