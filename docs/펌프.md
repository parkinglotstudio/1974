# 진화(Evolution) 시스템 — 테이블 구조화 기획서

## 개요
기존 "탈렌트(talents)" 시스템을 폐기하고, `evolution_config.csv` (tang SSoT) 기준의
**진화 노드(Evolution Node) 시스템**으로 전면 교체. 이번 작업부터 본 프로젝트의
모든 구조적/게임플레이 데이터는 **데이터 테이블(JSON/CSV 기반) 관리 원칙**을 따른다.
하드코딩된 구조 데이터는 두지 않는다.

## SSoT
- 원본: `C:\1974\games\tang\public\tables\evolution_config.csv`
- 포팅 결과: `C:\1974\games\pump\data\evolution_config.json`

## 데이터 구조

### evolution_config.json
```json
{
  "baseCost": 200,
  "costPerOrder": 100,
  "nodes": [
    { "id", "order", "branch", "name", "title", "desc", "icon", "abilityType", "effectValue", "prereq" }
  ]
}
```

- 총 11노드, 2개 분기(branch)
  - **branch 0** — 메인 8노드 직선 체인 (order 1~8, prereq = 이전 노드 id)
  - **branch 1** — 사이드 노드 3개 (id 101/102/103), branch0의 3/5/7번 노드에서 분기
- **1회 획득형** — 레벨업이 아닌 보유 여부(`true`/`false`)로 영구 효과 적용
- `cost = baseCost + order * costPerOrder`
- `abilityType` 종류:
  | type | 효과 |
  |---|---|
  | armor | 피해 감소량 누적 (`dmgReduce`) |
  | hp | 최대 체력 가산 (`hpAdd`) |
  | power | 데미지 배율 가산 (`dmgMult`) |
  | speed | 이동속도 배율 가산 (`speedMult`) |
  | cooldown | 스킬 쿨다운 배율 감소 (`cooldownMult`) |
  | magnet | 아이템 흡수 반경 배율 가산 (`magnetMult`) |
  | predator | 처치 시 확률적 HP 회복 (`lifesteal`) |

### save 데이터 (SaveSystem.js)
```js
evoNodes: { [evoNodeId]: true }  // 보유한 노드 id만 기록
```
기존 `save.talents` 필드는 완전 제거.

## 로직

### SaveSystem.computeBonuses(save, meta, evo)
- 장비(`meta.equipment` × `save.equipLevels`) 보너스 + 진화 노드(`evo.nodes` × `save.evoNodes`) 보너스를 합산
- 반환값: `{ dmgMult, speedMult, hpAdd, dmgReduce, cooldownMult, magnetMult, lifesteal }`

### PlayScene.js 연결
- `bonus.cooldownMult` → `SkillRunner` 5번째 인자(`extraCooldownMult`)로 전달, 모든 스킬 쿨다운에 곱연산
- `bonus.dmgReduce` → 적 피격 데미지에 `dmg * (1 - dmgReduce)` 적용
- `bonus.magnetMult` → `XpDrops.tick()`의 픽업 반경 배율로 전달
- `bonus.lifesteal` → 적 처치 시 `Math.random() < lifesteal` 확률로 `maxHp * 0.05` 회복
  - ⚠️ **임시값**: CSV에는 발동 확률(`effectValue=0.15`)만 정의되어 있고 회복량 수치는 없음.
    추후 회복량 필드가 CSV에 추가되면 그 값으로 교체 필요.

### LobbyScene.js — 진화 탭 UI 로직
- 기존 UI 레이아웃(`evo_name0~2`/`evo_desc0~2`/`evo_gauge0~2`/`evo_state0~2`, 3슬롯) 변경 없음
- `_evoFrontier()` — 분기(branch)별로 "획득 가능한(=prereq 충족 & 미보유) 최소 order 노드"를 1개씩 추출,
  최대 3개를 화면에 노출
- `_evoCost(node)` — `baseCost + order * costPerOrder`
- `_acquireEvoNode(i)` — 골드 차감 후 `save.evoNodes[node.id] = true`
