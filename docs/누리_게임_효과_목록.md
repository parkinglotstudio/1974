# 누리(Nuri) 게임 — 적용 효과 목록

> 기준 버전: NuriGame.js v2.8  
> 마지막 업데이트: 2026-06-17

---

## 1. 모션 트레일 (달리기 중)

**함수:** `_renderTrail(ctx, entity)`  
**발동:** `runState === 'running'` 동안 매 프레임

- 진행 반대 방향으로 5개의 ghost 잔상을 순서대로 렌더
- 뒤로 갈수록 alpha 감소 (앞 0.30 → 뒤 0.06)
- `globalCompositeOperation = 'lighter'` (가산 합성) → 어두운 배경에서 빛나는 잔광 느낌
- 잔상 간격: 7px, 현재 프레임의 `_entBuf`를 그대로 재사용해서 퍼포먼스 손실 없음

---

## 2. 발밑 모래 먼지 (달리기 중)

**함수:** `_updateDust(dt)` / `_renderDust(ctx)`  
**발동:** `runState === 'running'` 동안 매 프레임, 초당 약 18개 스폰

- 캐릭터 `_entBuf`의 하단 22% 픽셀에서 색상 샘플링 후 입자 생성
- 입자가 진행 반대 방향(-35px/s)으로 흩날리며 중력(120px/s²) 낙하
- 수명 0.4~0.9초, 수명 60% 이후부터 fade out
- 2×2 그리드 정렬로 픽셀아트 느낌 유지

---

## 3. 모래시계 스트림 (클릭 이동)

**함수:** `_renderPixelMove(ctx, b)`  
**발동:** 클릭 후 캐릭터 이동 시

캐릭터 픽셀이 4단계로 흘러서 목적지에 재조립되는 모래시계 이동 연출.

| 구간 | trav 범위 | 동작 |
|---|---|---|
| 출발 | 0 → 0.25 | 각 픽셀이 수집점(gatherX/Y)으로 모임 |
| 스트림 | 0.25 → 0.75 | 직선으로 방출점(releaseX/Y)까지 흐름 |
| 도착 | 0.75 → 1.0 | 목적지 형태로 재조립 (cubic ease) |

- 파티클별로 `ff`(출발 딜레이)가 랜덤 → 픽셀이 한꺼번에 움직이지 않고 물결처럼 흐름
- **거리 비례 duration:** 최소 350ms ~ 최대 1000ms (화면 폭 960px 기준 1400ms 비율)
  - 가까운 클릭 = 빠르게, 먼 클릭 = 느리게

---

## 4. 스파이크 색상 (클릭 이동 스트림 중)

**위치:** `_renderPixelMove` 스트림 구간 내부  
*(현재 비활성 — ECG 파형 제거로 같이 비활성됨)*

> ECG 파형이 튀는 순간 각 픽셀에 +140R, +180G, +255B 를 더해 파란색으로 번쩍이는 효과.  
> ECG 파형 복구 시 자동으로 함께 활성화됨.

---

## 5. 출발 / 도착 플래시 (클릭 이동)

**위치:** `_renderPixelMove` 내 플래시 블록  
**상수:** `FLASH_MS = 200`

- 이동 **시작** 200ms: 출발 위치 픽셀이 흰색으로 번쩍이며 사라짐
- 이동 **완료** 200ms: 도착 위치 픽셀이 흰색으로 번쩍이며 나타남

---

## 6. 오비터 (항상)

**함수:** `_renderOrbits(ctx, frontPass, activeEnt)`  
**발동:** 항상 (아이들·달리기·클릭이동 무관)

- 캐릭터 허리 높이(ph × 0.38) 기준으로 5개 오비터가 타원 궤도로 공전
- 각 오비터는 네모↔원형 사이를 `morphT`로 smoothstep 모프
- sin(angle) > 0 = 앞 패스, < 0 = 뒤 패스로 나눠서 캐릭터 앞뒤 입체감 표현
- `globalCompositeOperation = 'lighter'` 가산 합성, 핑크·시안 2색 교차

## 상수 요약

| 상수 | 값 | 설명 |
|---|---|---|
| `CHAR_SCALE` | 1.0 | 캐릭터 렌더 배율 (원본 해상도) |
| `RUN_SPEED` | 160 px/s | 조이스틱 달리기 속도 |
| `FLASH_MS` | 200 ms | 클릭 이동 플래시 지속 시간 |
| `WAIST_SCALE` | 0.05 | 스트림 허리 폭 계수 |
| `PM_DURATION` | — | 거리 비례로 동적 계산 (350~1000ms) |
